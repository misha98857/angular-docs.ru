# Управление потоком в компонентах - `@for`

При разработке веб-приложений часто возникает необходимость повторить определенный код несколько раз — например, имея
массив имен, вы можете захотеть отобразить каждое имя в теге `<p>`.

Примечание: Подробнее
об [управлении потоком читайте в руководстве по основам](/essentials/templates#control-flow-with-if-and-for).

В этом упражнении вы узнаете, как использовать `@for` для повторения элементов в шаблоне.

<hr/>

Синтаксис, позволяющий повторять элементы в шаблоне — это `@for`.

Вот пример использования синтаксиса `@for` в компоненте:

```angular-ts
@Component({
  ...
  template: `
    @for (os of operatingSystems; track os.id) {
      {{ os.name }}
    }
  `,
})
export class App {
  operatingSystems = [{id: 'win', name: 'Windows'}, {id: 'osx', name: 'MacOS'}, {id: 'linux', name: 'Linux'}];
}
```

Обратите внимание на два момента:

- Префикс `@` перед `for` используется потому, что это специальный синтаксис,
  называемый [синтаксисом шаблонов Angular](guide/templates).
- Для приложений версии 16 и старше, пожалуйста, обратитесь
  к [документации Angular по NgFor](guide/directives/structural-directives).

<docs-workflow>

<docs-step title="Добавьте свойство `users`">
В классе `App` добавьте свойство `users`, содержащее пользователей и их имена.

```ts
[{id: 0, name: 'Sarah'}, {id: 1, name: 'Amy'}, {id: 2, name: 'Rachel'}, {id: 3, name: 'Jessica'}, {id: 4, name: 'Poornima'}]
```

</docs-step>

<docs-step title="Обновите шаблон">
Обновите шаблон, чтобы отобразить имя каждого пользователя в элементе `p`, используя синтаксис шаблона `@for`.

```angular-html
@for (user of users; track user.id) {
  <p>{{ user.name }}</p>
}
```

ПРИМЕЧАНИЕ: использование `track` обязательно; вы можете использовать `id` или любой другой уникальный идентификатор.

</docs-step>

</docs-workflow>

Этот тип функциональности называется управлением потоком (control flow). Далее вы научитесь настраивать компоненты и
взаимодействовать с ними — кстати, вы отлично справляетесь.
