# Suffix not supported

Эта диагностика срабатывает, когда суффиксы `.px`, `.%` и `.em` используются при привязке к атрибуту.

```html
<img [attr.width.px]="5">
```

## В чем проблема?

Эти суффиксы доступны только для привязки стилей. Они не имеют смысла при привязке к атрибуту.

## Что следует сделать вместо этого?

Вместо использования суффиксов `.px`, `.%` или `.em`, которые поддерживаются только в привязке стилей, укажите единицы
измерения непосредственно в значении привязки.

```html
<img [attr.width]="'5px'">
```

## Требования к конфигурации

Для работы любой расширенной диагностики необходимо включить опцию [
`strictTemplates`](tools/cli/template-typecheck#strict-mode).
`suffixNotSupported` не имеет дополнительных требований, кроме `strictTemplates`.

## Что если я не могу этого избежать?

Эту диагностику можно отключить, отредактировав файл `tsconfig.json` проекта:

```json

{
  "angularCompilerOptions": {
    "extendedDiagnostics": {
      "checks": {
        "suffixNotSupported": "suppress"
      }
    }
  }
}
```

Подробнее см. в разделе [конфигурация расширенной диагностики](extended-diagnostics#configuration).
