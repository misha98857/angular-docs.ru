# Функции должны вызываться при интерполяции текста

Эта диагностика обнаруживает невызванные функции в интерполяции текста.

```angular-ts
import { Component } from '@angular/core';

@Component({
  template: `{{ getValue }}`,
})
class MyComponent {
  getValue() {
    return 'value';
  }
}
```

## В чем проблема?

Функции в интерполяции текста должны вызываться для возврата значения.
Если функция не вызвана, она не вернет никакого значения, и интерполяция будет работать не так, как ожидается.

## Что следует сделать?

Убедитесь, что вы вызываете функцию при использовании ее в интерполяции текста, чтобы вернуть значение.

```angular-ts
import { Component } from '@angular/core';

@Component({
  template: `{{ getValue() }}`,
})
class MyComponent {
  getValue() {
    return 'value';
  }
}
```

## Требования к конфигурации

Для работы любой расширенной диагностики необходимо включить [
`strictTemplates`](/tools/cli/template-typecheck#strict-mode).
У `uninvokedFunctionInTextInterpolation` нет дополнительных требований, кроме `strictTemplates`.

## Что делать, если этого нельзя избежать?

Эту диагностику можно отключить, отредактировав файл `tsconfig.json` проекта:

```json
{
  "angularCompilerOptions": {
    "extendedDiagnostics": {
      "checks": {
        "uninvokedFunctionInTextInterpolation": "suppress"
      }
    }
  }
}
```

См. [конфигурацию расширенной диагностики](/extended-diagnostics#configuration) для получения дополнительной информации.
