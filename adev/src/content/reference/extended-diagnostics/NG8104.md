# Текстовый атрибут без привязки

Эта диагностика проверяет, что атрибуты, имеющие «специальный» префикс привязки Angular (`attr.`, `style.` и `class.`),
интерпретируются именно как привязки.

```html
<div attr.id="my-id"></div>
```

## В чем проблема?

В этом примере `attr.id` интерпретируется как обычный атрибут и попадет в итоговый HTML в неизменном виде (
`<div attr.id="my-id"></div>`). Скорее всего, это не входило в планы разработчика. Вместо этого целью, вероятно, была
установка атрибута `id` (`<div id="my-id"></div>`).

## Что следует сделать?

При привязке к `attr.`, `class.` или `style.` используйте синтаксис привязки шаблонов Angular (`[]`).

```html
<div [attr.id]="my-id"></div>
<div [style.color]="red"></div>
<div [class.large]="true"></div>
```

## Требования к конфигурации

Для работы любой расширенной диагностики необходимо включить опцию [
`strictTemplates`](tools/cli/template-typecheck#strict-mode). У `textAttributeNotBinding` нет дополнительных требований,
кроме включенного `strictTemplates`.

## Что делать, если этого нельзя избежать?

Эту диагностику можно отключить, отредактировав файл `tsconfig.json` проекта:

```json
{
  "angularCompilerOptions": {
    "extendedDiagnostics": {
      "checks": {
        "textAttributeNotBinding": "suppress"
      }
    }
  }
}
```

Подробнее см. в разделе [конфигурация расширенной диагностики](extended-diagnostics#configuration).
