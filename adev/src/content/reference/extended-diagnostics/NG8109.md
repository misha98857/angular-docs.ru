# Сигналы должны вызываться в интерполяциях шаблонов

Эта диагностика обнаруживает сигналы, которые не были вызваны в интерполяциях шаблонов.

```angular-ts
import { Component, signal, Signal } from '@angular/core';

@Component({
  template: `<div>{{ mySignal }}</div>`,
})
class MyComponent {
  mySignal: Signal<number> = signal(0);
}
```

## В чем проблема?

Сигналы Angular представляют собой функции без аргументов (`() => T`). При выполнении они возвращают текущее значение
сигнала. Это означает, что для отображения значения в интерполяциях шаблонов их необходимо вызывать.

## Что нужно сделать?

Убедитесь, что вы вызываете сигнал при его использовании в интерполяции шаблона для отображения значения.

```angular-ts
import { Component, signal, Signal } from '@angular/core';

@Component({
  template: `<div>{{ mySignal() }}</div>`,
})
class MyComponent {
  mySignal: Signal<number> = signal(0);
}
```

## Требования к конфигурации

Для работы любой расширенной диагностики необходимо включить опцию [
`strictTemplates`](tools/cli/template-typecheck#strict-mode). У `interpolatedSignalNotInvoked` нет дополнительных
требований, кроме `strictTemplates`.

## Что делать, если этого нельзя избежать?

Эту диагностику можно отключить, изменив файл `tsconfig.json` проекта:

```json

{
  "angularCompilerOptions": {
    "extendedDiagnostics": {
      "checks": {
        "interpolatedSignalNotInvoked": "suppress"
      }
    }
  }
}
```

Подробнее см. в разделе [конфигурация расширенной диагностики](extended-diagnostics#configuration).
