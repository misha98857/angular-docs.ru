# Функции должны вызываться в привязках событий

Эта диагностика обнаруживает невызванные функции в привязках событий.

```typescript
import { Component, signal, Signal } from '@angular/core';

@Component({
  template: `<button (click)="(onClick)">Click me</button>`,
})
class MyComponent {
  onClick() {
    console.log('clicked');
  }
}
```

## В чем ошибка?

Функции в привязках событий должны вызываться при срабатывании события.
Если функция не вызвана, она не выполнится при срабатывании события.

## Что следует сделать вместо этого?

Убедитесь, что вы вызываете функцию при использовании её в привязке события, чтобы выполнить функцию при срабатывании
события.

```typescript
import { Component } from '@angular/core';

@Component({
  template: `<button (click)="onClick()">Click me</button>`,
})
class MyComponent {
  onClick() {
    console.log('clicked');
  }
}
```

## Требования к конфигурации

Для работы любой расширенной диагностики необходимо включить [
`strictTemplates`](tools/cli/template-typecheck#strict-mode).
У `uninvokedFunctionInEventBinding` нет дополнительных требований, кроме `strictTemplates`.

## Что делать, если этого нельзя избежать?

Эту диагностику можно отключить, отредактировав файл `tsconfig.json` проекта:

```json

{
  "angularCompilerOptions": {
    "extendedDiagnostics": {
      "checks": {
        "uninvokedFunctionInEventBinding": "suppress"
      }
    }
  }
}
```

См. [конфигурацию расширенной диагностики](extended-diagnostics#configuration) для получения дополнительной информации.
