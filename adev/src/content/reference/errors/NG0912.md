# Коллизия при генерации ID компонентов

При создании компонентов Angular генерирует уникальный ID для каждого из них. Он создается на основе метаданных
компонента Angular, включая, помимо прочего: селекторы, количество привязок хоста (host bindings), имена свойств класса,
запросы к представлению и содержимому (view и content queries). Когда метаданные двух компонентов идентичны (часто при
использовании одного и того же селектора), происходит коллизия (конфликт) генерации ID.

ID компонентов используются внутри Angular:

- для дополнительных аннотаций DOM-узлов с целью инкапсуляции стилей;
- во время [гидратации](guide/hydration) для восстановления состояния приложения
  после [рендеринга на стороне сервера (SSR)](guide/ssr).

Чтобы избежать проблем, которые могут быть вызваны коллизией ID компонентов, рекомендуется устранить их, как описано
ниже.

## Пример коллизии ID компонентов

```typescript
@Component({
  selector: 'my-component',
  template: 'complex-template',
})
class SomeComponent {}

@Component({
  selector: 'my-component',
  template: 'empty-template',
})
class SomeMockedComponent {}
```

Наличие этих двух определенных компонентов вызовет коллизию генерации ID, и во время разработки будет отображено
предупреждение.

## Отладка ошибки

Сообщение с предупреждением содержит имена классов двух компонентов, чьи ID конфликтуют.

Проблему можно решить, используя одно из приведенных ниже решений:

1. Измените селектор одного из двух компонентов. Например, используя псевдоселектор, не влияющий на результат, такой как
   `:not()`, и другое имя тега.

```typescript
@Component({
  selector: 'my-component:not(p)',
  template: 'empty-template',
})
class SomeMockedComponent {}
```

1. Добавьте дополнительный host-атрибут к одному из компонентов.

```typescript
@Component({
  selector: 'my-component',
  template: 'complex-template',
  host: {'some-binding': 'some-value'},
})
class SomeComponent {}
```
