# Поведение @defer при включенном HMR

Hot Module Replacement (HMR) — это технология, используемая серверами разработки для предотвращения перезагрузки всей
страницы при изменении только части приложения.

Когда в Angular включен HMR, все зависимости блока `@defer` загружаются сразу (eagerly), не дожидаясь настроенных
условий срабатывания (это касается как триггеров только для клиента, так и триггеров инкрементальной гидратации). Это
необходимо для корректной работы HMR, который заменяет компоненты приложения во время выполнения без перезагрузки всей
страницы. ПРИМЕЧАНИЕ: фактический рендеринг блоков defer в режиме HMR по-прежнему зависит от условий триггеров.

Если вы хотите протестировать поведение блока `@defer` в режиме разработки и убедиться, что необходимые зависимости
загружаются только при выполнении условия триггера, вы можете отключить режим HMR, как описано в [
`этом документе`](/tools/cli/build-system-migration#hot-module-replacement).
