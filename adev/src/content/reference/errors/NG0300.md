# Конфликт селекторов

<docs-video src="https://www.youtube.com/embed/z_3Z5mOm59I"/>

Два или более [компонента](guide/components) используют один и тот же селектор элемента. Поскольку с одним элементом
может быть связан только один компонент, селекторы должны быть уникальными строками, чтобы избежать неоднозначности для
Angular.

Эта ошибка возникает во время выполнения, когда вы применяете два селектора к одному узлу, каждый из которых
соответствует разным компонентам, или когда вы применяете один селектор к узлу, и он соответствует более чем одному
компоненту.

```ts
import {Component} from '@angular/core';

@Component({
  selector: '[stroked-button]',
  templateUrl: './stroked-button.component.html',
})
export class StrokedBtnComponent {}

@Component({
  selector: '[raised-button]',
  templateUrl: './raised-button.component.html',
})
export class RaisedBtnComponent {}

@Component({
  selector: 'app-root',
  template: `

  <!-- У этого узла 2 селектора: stroked-button и raised-button, и оба соответствуют разным компонентам: StrokedBtnComponent и RaisedBtnComponent, поэтому возникнет ошибка NG0300 -->

<button stroked-button  raised-button></button>
`,
})
export class AppComponent {}
```

## Отладка ошибки

Используйте имя элемента из сообщения об ошибке для поиска мест в кодовой базе, где используется объявление такого же
селектора:

```ts

@Component({
  selector: 'YOUR_STRING',
…
})

```

Убедитесь, что каждый компонент имеет уникальный CSS-селектор. Это гарантирует, что Angular отрендерит именно тот
компонент, который вы ожидаете.

Если вам трудно найти несколько компонентов с таким именем тега селектора, проверьте компоненты из импортированных
библиотек, таких как Angular Material. Убедитесь, что вы следуете [лучшим практикам](style-guide#component-selectors)
именования селекторов, чтобы предотвратить конфликты.
