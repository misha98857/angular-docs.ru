# Заголовки не переданы через HttpTransferCache

Angular выводит это предупреждение, когда код пытается получить доступ к HTTP-заголовку, который не был передан с
сервера на клиент с помощью `HttpTransferCache`.

По умолчанию `HttpTransferCache` не передает никакие заголовки, чтобы избежать потенциальных проблем с безопасностью или
ненужной передачи данных. Когда на клиенте происходит обращение к заголовку, который не был включен в передачу, Angular
выводит это предупреждение, чтобы помочь вам определить отсутствующие заголовки.

## Исправление ошибки

Чтобы включить конкретные заголовки в кэш передачи (transfer cache), используйте опцию `includeHeaders`.

### На уровне запроса

Добавьте параметр `transferCache` к вашему HTTP-запросу:

```typescript
this.http.get('/api/data', {
  transferCache: {
    includeHeaders: ['cache-control', 'etag']
  }
});
```

### На уровне приложения

Настройте `includeHeaders` при подключении `HttpTransferCache`:

```typescript
provideClientHydration(
  withHttpTransferCache({
    includeHeaders: ['cache-control', 'etag']
  })
);
```

Включайте только те заголовки, которые необходимы для логики вашего приложения. Будьте осторожны с заголовками,
содержащими конфиденциальные данные.

## Отладка ошибки

Сообщение с предупреждением содержит имя заголовка и URL запроса. Проверьте в коде места, где происходит обращение к
этому заголовку, и определите, нужно ли включать его в кэш передачи.

Если заголовок не нужен на клиенте, рассмотрите возможность рефакторинга кода, чтобы избежать обращения к нему.
