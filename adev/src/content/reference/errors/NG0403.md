# Загружаемый NgModule не указывает, какой компонент инициализировать

Эта ошибка означает, что в NgModule, который использовался для загрузки приложения, отсутствует ключевая информация,
необходимая Angular для продолжения процесса загрузки.

Ошибка возникает, когда свойство `bootstrap` отсутствует (или является пустым массивом) в декораторе `@NgModule` и в
этом классе NgModule не определен хук жизненного цикла `ngDoBootstrap`.

Дополнительную информацию о процессе загрузки можно найти в [этом руководстве](guide/ngmodules/bootstrapping).

Следующие примеры вызовут эту ошибку.

```typescript
@NgModule({
  declarations: [AppComponent],
  imports: [BrowserModule, AppRoutingModule],
  providers: [],
})
export class AppModule {}

// AppModule используется для загрузки, но поле `@NgModule.bootstrap` отсутствует.
platformBrowser().bootstrapModule(AppModule);
```

```typescript
@NgModule({
  declarations: [AppComponent],
  imports: [BrowserModule, AppRoutingModule],
  providers: [],
  bootstrap: [],
})
export class AppModule {}

// AppModule используется для загрузки, но поле `@NgModule.bootstrap` содержит пустой массив.
platformBrowser().bootstrapModule(AppModule);
```

## Отладка ошибки

Пожалуйста, убедитесь, что NgModule, используемый для загрузки, настроен правильно:

- либо свойство `bootstrap` существует (и содержит непустой массив) в декораторе `@NgModule`
- либо метод `ngDoBootstrap` существует в классе NgModule
