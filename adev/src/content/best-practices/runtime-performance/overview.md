# Оптимизация производительности во время выполнения

Быстрый рендеринг имеет решающее значение для Angular, и мы создали фреймворк с учетом множества оптимизаций, чтобы
помочь вам разрабатывать производительные приложения. Чтобы лучше понять производительность вашего приложения, мы
предлагаем [Angular DevTools](tools/devtools) и [видео-руководство](https://www.youtube.com/watch?v=FjyX_hkscII) по
использованию Chrome DevTools для профилирования. В этом разделе мы рассмотрим наиболее распространенные методы
оптимизации производительности.

**Обнаружение изменений** (Change detection) — это процесс, с помощью которого Angular проверяет, изменилось ли
состояние вашего приложения и нужно ли обновлять DOM. На высоком уровне Angular обходит ваши компоненты сверху вниз в
поисках изменений. Angular периодически запускает механизм обнаружения изменений, чтобы изменения в модели данных
отражались в представлении приложения. Обнаружение изменений может быть запущено вручную или посредством асинхронного
события (например, взаимодействие с пользователем или завершение XMLHttpRequest).

Обнаружение изменений сильно оптимизировано и производительно, но оно все же может вызывать замедления, если приложение
запускает его слишком часто.

В этом руководстве вы узнаете, как контролировать и оптимизировать механизм обнаружения изменений, исключая части вашего
приложения из проверки и запуская обнаружение изменений только при необходимости.
